<Aside type="tip" title="Why you need this">
The moment you run more than one agent â€” a USER tier vs an ADMIN tier, a customer-facing bot vs an
internal one â€” "every tool, always on" is wrong and often unsafe. A support agent should not see

by code, same semantics in every port.

Four controls, smallest blast radius first:

## 1. Allow only specific tools from an MCP server

In `mcp.json`, a per-server `tools` map is an **allowlist** when it has any `true` entry (only those
load), a **drop-list** when it has only `false` entries, and "all tools" when absent/empty. Keys are
the server's original tool names. This is config, identical across all six ports:

```json
{
  "mcpServers": {
    "hr": {
      "url": "https://hr.internal/mcp",
      "tools": { "get_leave_balance": true, "get_profile": true }
    },
    "files": {
      "command": ["npx", "-y", "@modelcontextprotocol/server-filesystem", "/data"],
      "tools": { "delete_file": false }
    }
  }
}
```

The `hr` server exposes **only** those two tools; `files` exposes everything **except**
`delete_file`.

## 2. Turn built-ins off, or pick specific ones

The ten built-in tools are on by default. Turn the whole source off, or drop individual ones with a
nameâ†’bool map (Â§4A). Shown as a config block (shared) and as a toolkit option:

```json
{ "builtins": { "tools": { "bash": false, "write": false } } }
```

## 3. Drop any tool or skill by name â€” the "just turn these off" list

`disableTools` drops tools by their **final exposed name** across every source (MCP, builtins,
native, HTTP, A2A); `disableSkills` removes skills from the `skill` catalog. The simplest control,
and the one you reach for most:

```js

  disableSkills: ["deploy-prod"],            // remove a skill from the catalog
```

## 4. Allowlist skills per agent

The skill analog of the MCP `tools` map: `skillsFilter` (`skills_filter` in Go/Elixir) is a
nameâ†’bool allowlist. `{ "refunds": true }` exposes only that skill; only-`false` entries are a
drop-list; absent â‡’ all. Use it to give each agent tier its own skill set from one folder.

Together these compose: an MCP allowlist narrows a server, `builtins` scopes the shell/file tools,
`disableTools`/`disableSkills` trims anything by name, and `skillsFilter` picks the skill set â€” so
one codebase serves a locked-down USER agent and a full-power ADMIN agent from the same fixtures.
