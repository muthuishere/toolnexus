The two headline dynamic sources. Point the toolkit at an `mcp.json` (local stdio
**and** remote streamable-HTTP servers) and a folder of `**/SKILL.md` skills — both
join the same tool set. A failing MCP server is isolated; it never breaks the build.

<Tabs syncKey="lang">
<TabItem label="JavaScript">
```js
const tk = await createToolkit({
  mcpConfig: "./mcp.json",        // path, raw JSON string, or a parsed object
  skillsDir: "./skills",          // one or more skill roots
})
console.log(tk.tools().map((t) => `${t.name} (${t.source})`))
```
</TabItem>

An `mcp.json` looks like this (both server kinds shown):

```json
{
  "mcpServers": {
    "filesystem": {
      "command": ["npx", "-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
    },
    "remote-api": {
      "url": "https://example.com/mcp",
      "headers": { "Authorization": "Bearer ${MCP_TOKEN}" }
    }
  }
}
```

`${MCP_TOKEN}` expands from the environment at call time and is never logged. See
[MCP servers](/toolnexus/mcp/) and [Agent skills](/toolnexus/skills/) for the full
config surface.
